== README

Тестовое задание от Stone Labs http://stone-labs.com/

Основное задание:
В системе есть 2 роли – Пользователь и Админ

Админу доступно:
1. Активирует новых Пользователей
2. CRUD (create, read, update, delete) вопросов;
3. Создает и редактирует категории вопросов (подкатегории в дополнительном задании).
4. CRUD пользователей (в том числе изменение пароля пользователя).
5. Просмотр списка пользователей с количеством отвеченных ими вопросов.

Пользователю доступно:
1. Регистрация в системе - имя, емэйл и пароль. Доступ в систему после активации аккаунта Администратором. До активации Пользователь видит системное сообщение о том, что он ещё не активирован.
2. Вход в систему для активированных Админом Пользователей.
3. У каждого Пользователя есть статус (новичок, пользователь, доверенный, vip). По умолчанию, после регистрации Пользователь получает статус – новичок. Затем Админ может вручную менять статус Пользователя   на странице редактирования данных Пользователя.
4. После входа в систему Пользователь видит список вопросов. На каждый вопрос он может ответить. Ответ - это текст (одно или несколько слов). Для Пользователя отвеченные и не отвеченные вопросы отображаются по-разному.
5. Пользователь может просматривать вопросы: только отвеченные/только не отвеченные /все вопросы.

Дополнительное задание:
Все вопросы делятся на категории.
Некоторые категории содержат вложенные подкатегории (например, «Простые вопросы» -> «Вопросы личного характера» -> «Как зовут вашего питомца»). Каждый вопрос относится к какой-либо одной подкатегории.
На странице создания/редактирования вопроса Админ может изменить к какой подкатегории относится вопрос.

Нефункциональные требования:
1. Реализация кастомного дизайна необязательна (можно использовать  bootstrap), однако использование фреймворков
( вроде http://infinite-woodland-5276.herokuapp.com/index.html ) очень приветствуется.
2. Весь код должен быть хорошо документирован и понятен для 3х лиц.
Срок выполнения задания – 2 дня с момента заполнения формы.

Что необходимо предоставить:
1. Рабочую систему на Heroku или любом другом хостинге
2. Исходный код проекта на Github
3. Всю информацию присылайте по адресу wanted@stone-labs.com
